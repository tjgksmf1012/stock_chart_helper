# 정직한 안내 2종 — 모순 경고 배지 + 오늘의 최우선 카드

날짜: 2026-07-18
상태: 승인됨 (사용자 선택: "모순 경고 배지" + "오늘의 최우선 카드", 009240 한샘 사례에서 출발)

## 배경

한샘(009240): 패턴 상승확률 64.9% vs 전망 구간 1개월 중앙값 −1.1% — 두 숫자가
서로 다른 질문(단기 조건부 패턴 vs 무조건부 종목 체질)에 답하고 있어 충돌한다.
사용자가 이 충돌을 스스로 발견해야 하는 것이 문제. 또한 "지금 가장 들어갈 만한
후보"가 화면에서 명시적으로 지목되지 않는다. 단, 상승확률 순 추천 랭킹은 만들지
않는다 — 실측 엣지 +0.5%/거래로 판명된 지표로 줄 세우면 예전의 거짓말 기계로
돌아간다. 랭킹은 검증된 기대값(EV) 기준만 쓴다.

## ① 모순 경고 배지 (차트 페이지)

- 순수 함수 `detectOutlookConflict(pUp, horizons)` (frontend `lib/outlookConflict.ts`, vitest):
  - 충돌 = `pUp >= 0.58` 이고 1개월(20일) 구간의 `q50 < 0`
    (1개월 없으면 1주(5일)로 대체, 둘 다 없으면 충돌 아님. pUp이 null이면 아님)
  - 반환: `{ conflict: boolean, medianPct: number | null }`
- 렌더: `OutlookCard`에 옵션 prop `pUp` 추가 (ChartPage가 `analysis.p_up` 전달).
  충돌이면 카드 상단에 호박색 배지:
  "단기 패턴은 위(상승확률 N%)를 가리키지만, 이 종목의 과거 분포는 1개월 중앙값
  M%로 아래를 가리킵니다. 이런 충돌 구간은 패턴 신뢰를 낮춰 보수적으로 보는 편이
  안전합니다."
- 역방향(확률 낮음 + 분포 위)은 만들지 않는다 — 롱 온리 앱에서 위험한 방향은
  "패턴↑ 체질↓" 하나다 (YAGNI).

## ② 오늘의 최우선 카드 (오늘 탭)

- 백엔드: `/lab/signals` 응답의 `eligible_strategies` 항목에 `ev_pct` 추가
  (리포트에서 읽음 — 이미 로드돼 있음). 캐시 키 v3.
- 순수 함수 `pickTopSignal(signals, eligible)` (frontend `lib/topPick.ts`, vitest):
  - 후보 = verdict가 `pass`인 신호만 (watch는 최우선으로 지목하지 않는다 —
    관찰 등급을 "오늘의 추천"으로 부르는 순간 정직함이 깨진다)
  - 랭킹: 전략 EV 내림차순 → 신호일 최신순 → 코드 오름차순(결정적)
  - 없으면 null
- 렌더: `LiveSignals` 상단에 하이라이트 스트립 —
  "오늘의 최우선 — {전략 라벨} (검증 EV +N%/거래)": 종목 코드, 기준가, 손절가,
  권장 수량(기존 계산기 설정 재사용), [차트에서 확인] 버튼.
  pass 신호가 없으면: "오늘은 통과 등급 신호가 없습니다 — 안 사는 것도 전략입니다."
  (신호 자체가 0건일 때의 기존 빈 상태와는 별개 — 스트립은 신호가 있으되 전부
  watch일 때도 이 문구를 보여준다.)

## 하지 않는 것

- 상승확률 순 종목 추천 랭킹 (관찰 후보의 정렬 옵션으로 이미 존재하는 것 이상 금지)
- 역방향 충돌 경고, 종목명 조회 추가 API 호출

- pytest: eligible_strategies의 ev_pct 포함
- 브라우저: 009240 차트에서 충돌 배지 확인, 오늘 탭에서 최우선 카드/빈 문구 확인
