# 증거 우선 트레이딩 랩 (Evidence-First Trading Lab) 설계

날짜: 2026-07-12
상태: 승인됨 (사용자 확인)

## 배경 — 왜 방향을 바꾸나

2026-07 전체 앱 리뷰와 후속 수정 과정에서, 이 앱이 스스로 측정해 둔 데이터가
현재 추천 방식의 한계를 보여줬다:

1. **학습 모델의 판정**: 실측 1,531건으로 학습한 로지스틱 회귀
   (`backend/data/probability_model.json`)의 계수가 대부분 0 근처 또는 음수 —
   규칙 점수·유사 사례 승률·손익비 등 핵심 특징이 실제 승패를 거의 못 가른다.
2. **기저율 30%**: 확정 패턴의 목표가 선터치 승률이 ~30%인데, 평균 성공 수익
   (+6~11%)이 평균 실패 손실(-10~19%)보다 작아 거래당 기대값이 음수 방향.
3. **개발 이력의 자인**: probability_model.py 주석 — 캘리브레이션의 brier
   score가 "그냥 base_rate 고정 예측"보다 나빴다.
4. **구조적 문제**: 백테스트가 현재 시총 상위 유니버스를 과거로 돌림(생존
   편향), 파라미터를 같은 데이터로 튜닝(인샘플), 거래 비용 미반영, 숏 추천은
   한국 개인투자자가 실행 불가.

결론: 신호 생성기를 더 정교하게 만드는 방향이 아니라, **"검증을 통과하지 못한
신호는 화면에 올리지 않는" 검증 파이프라인**을 코어로 놓고, 문헌상 근거 있는
전략들을 그 위에서 판정하는 방향으로 전환한다.

## 제품 철학

> 이 앱의 약속은 "좋은 신호를 준다"가 아니라
> **"검증을 통과하지 못한 신호는 화면에 올라오지 않는다"**이다.

- 모든 추천에는 검증 근거가 붙는다: 검증 구간, 비용 차감 후 거래당 EV,
  95% 신뢰구간, 표본 수.
- 어떤 전략도 통과하지 못하면 대시보드는 "현재 검증 통과 전략 없음"을
  보여준다. 이것은 실패 상태가 아니라 제품이 정직하게 작동하는 모습이다.
- 합성 점수(우위 %, 상승 확률 %)는 추천 근거에서 퇴출한다. 실측 EV와
  신뢰구간만 추천 근거가 된다.
- 숏 방향 추천은 제거한다 (한국 개인 공매도 실행 불가). 하락 신호는
  "매수 회피 / 보유분 점검" 정보로만 쓴다.

## 접근 방식 결정

**기존 레포 안에서 코어 교체** (그린필드 재작성 기각):

- 유지: 데이터 수집기(pykrx 쿨다운/FDR 폴백), 판단 저널·결과 추적·캘리브레이션
  루프, UI 셸(React/차트/관심종목), 스캔 인프라(배치·취소·진행률).
- 신설: `backend/app/lab/` (검증 엔진), `backend/app/strategies/` (전략들).
- 강등: 기존 패턴 엔진은 삭제하지 않고 (a) 차트 위 참고용 주석, (b) 새 검증
  파이프라인의 첫 피검체로 사용.
- 시간축: 스윙(며칠~몇 주), 일봉 기준. 파라미터로 시간축 변경 가능하게 설계.

## 아키텍처

### backend/app/lab/ — 검증 엔진 (Phase 1)

```
lab/
  universe.py    # 시점 고정(point-in-time) 유니버스
  costs.py       # 거래 비용 모델
  walkforward.py # 워크포워드 하네스
  metrics.py     # 성과 지표 + 부트스트랩 CI + 판정
```

**universe.py** — pykrx `get_market_ticker_list(날짜)`로 "그 시점의 상장
종목"을 구성한다. 리밸런스 시점마다 당시 유니버스를 다시 뽑아 생존 편향을
제거한다. 과거 시세를 못 받아온 종목은 버리되, 리포트에 데이터 커버리지 %로
명시한다. 유니버스 규칙(예: 시총 상위 N, 거래대금 하한)은 전략 스펙의 일부다.

**costs.py** — 수수료 + 거래세 + 슬리피지. 보수적 기본값: 왕복 0.35%
(설정으로 조정 가능). 모든 수익률 지표는 비용 차감 후 값만 존재한다.
비용 차감 전 수치는 어디에도 표시하지 않는다.

**walkforward.py** — 롤링 학습/검증 분리. 기본: 학습 2년 → 검증 6개월 →
6개월 전진 반복. 전략 파라미터는 학습 구간에서만 선택하고, 검증 구간은 한 번만
본다. 인샘플(학습 구간) 성적은 내부 진단용으로만 저장하고 화면에는 절대
노출하지 않는다. 같은 종목에서 포지션이 겹치지 않게 1종목 1포지션 원칙.

**metrics.py** — 검증 구간 트레이드 목록에서 계산:
- 거래당 EV(비용 차감), 승률, 손익비, 최대 낙폭(MDD), 노출도, 표본 수
- 벤치마크 대비: KOSPI 매수보유, 그리고 "랜덤 진입 + 동일 청산 규칙"
  (귀무가설 — 이걸 못 이기면 청산 규칙이 좋은 것이지 진입에 엣지가 없는 것)
- 부트스트랩(트레이드 리샘플링) 95% 신뢰구간
- **판정 3등급**:
  - `통과`: 검증 구간 EV > 0이고 95% CI 하한 > 0이며, EV가 랜덤 진입
    벤치마크(동일 청산 규칙)의 평균 EV보다 큼
  - `관찰`: EV > 0이나 CI에 0이 포함되거나 랜덤 벤치마크를 못 넘음
  - `탈락`: EV ≤ 0

### backend/app/strategies/ — 전략 (Phase 2)

전략은 선언적 스펙을 구현한 파일 하나:

```python
class Strategy(Protocol):
    id: str
    label: str
    universe_rule: UniverseRule          # 시총/거래대금 필터
    def fit(self, train_bars) -> Params  # 학습 구간에서 파라미터 선택 (없으면 고정값)
    def signals(self, bars, params) -> list[Signal]
    # Signal = 진입일, 진입가, 손절, 목표(옵션), 시간 청산(옵션), 방향(long 고정)
```

피검체 목록 (전부 롱 온리, 일봉):

| id | 전략 | 근거 |
|---|---|---|
| xs_momentum | 횡단면 모멘텀 (12-1개월, 상위 분위 보유) | 학계 최다 반복 검증 |
| high52_breakout | 52주 신고가 근접 돌파 | George & Hwang 계열, 규칙 단순 |
| vol_breakout | 변동성 돌파 (k-팩터) | 기계적 진입·청산 |
| trend_momentum | 추세 필터(가격>이평) + 모멘텀 결합 | 추세추종 최소형 |
| legacy_patterns | 기존 패턴 엔진 confirmed top-N | 같은 하네스에서 공정 판정 |
| random_entry | 랜덤 진입 + 동일 청산 (벤치마크) | 귀무가설 |

### 데이터 (신규 테이블)

- `lab_runs`: 전략 id, 검증 설정(기간·비용·유니버스), 판정, 요약 지표, 실행 시각
- `lab_trades`: run별 시뮬레이션 트레이드 (진입/청산 일자·가격, 수익률, 청산 사유)
- 판정 결과는 기존 캐시(메모리/Redis)에도 올려 대시보드가 읽는다.

### 제품 통합 (Phase 3)

- **대시보드 개편**: 최상단이 "전략 성적표" — 통과/관찰 전략 카드(검증 성적 +
  오늘의 신호 목록). 탈락 전략은 실험실 페이지에서 사유와 함께 열람.
- **신호 게이트**: 통과/관찰 전략의 신호만 추천으로 노출. 관찰 등급은 명시적
  경고 라벨. 탈락 전략의 신호는 API에서도 추천으로 반환하지 않는다.
- **자동 종이매매**: 통과/관찰 전략의 신호는 기존 판단 저널(signal_outcomes)에
  자동 기록되고 기존 자동 평가 루프가 닫는다. 사용자 클릭 없이 실측 성적이
  쌓인다. 실측은 전략 id로 태깅해 전략별 실측 성적표를 만든다.
- **실험실 페이지** (신규): 전략별 검증 리포트 — 워크포워드 구간별 성적,
  트레이드 분포, 벤치마크 비교, 데이터 커버리지, 판정 사유.
- 차트 페이지·관심종목·저널·시스템 상태는 유지. 패턴 오버레이는 참고용으로
  남고 확률 배지는 제거.

### 드리프트 감시 (Phase 4)

- 실측(종이매매) 성적이 쌓이면 백테스트 대비 실측의 괴리를 감시한다.
  실측 EV가 백테스트 CI 하한을 이탈하면 전략을 자동으로 `관찰`로 강등하고
  사유를 표시한다.

## 하지 않는 것 (비목표)

- 실거래 주문 연동 (시세 조회 전용 유지)
- 숏/파생 전략
- 분봉 기반 단타 전략 (데이터 품질 확보 전까지)
- LLM 기반 추천
- 기존 패턴 엔진 삭제 (강등만)

## 정직한 한계 (제품에 명시할 것)

- 후보 전략 전부가 탈락할 수 있다. 그 경우 "지금 검증된 엣지 없음"을 보여주는
  것이 이 제품의 정상 동작이며, 그것이 손실을 막는 가치다.
- 무료 데이터의 한계(상폐 종목 시세 누락 가능성)는 커버리지 %로 공개한다.
- 검증 통과가 미래 수익을 보장하지 않는다. 드리프트 감시가 그 간극을 다룬다.
- 이 앱은 투자 권유 서비스가 아니라 분석·검증 보조 도구다 (기존 면책 유지).

## 단계별 계획

1. **Phase 1 — 랩 코어**: universe/costs/walkforward/metrics 4모듈(TDD) +
   기존 패턴 엔진을 첫 피검체로 돌린 검증 리포트 API/CLI.
   완료 기준: "기존 시스템의 비용 차감 후 워크포워드 성적"이 숫자로 나온다.
2. **Phase 2 — 전략 후보**: 전략 4종 + 랜덤 벤치마크 구현, 실험실 페이지.
3. **Phase 3 — 신호 게이트**: 대시보드 개편, 통과 전략 신호만 노출,
   자동 종이매매 저널링.
4. **Phase 4 — 드리프트 감시**: 실측 vs 백테스트 괴리 자동 강등.

각 Phase는 독립적으로 배포 가능하고, Phase 1만으로도 "현 시스템 판정"이라는
가치가 나온다.

## 테스트 전략

- lab 모듈은 전부 순수 함수 중심으로 설계해 합성 시세로 단위 테스트 (TDD).
- 워크포워드 하네스: 미래 데이터 누출(look-ahead) 회귀 테스트 — 검증 구간
  데이터가 파라미터 선택에 쓰이면 실패하는 테스트를 명시적으로 둔다.
- 비용 모델: 왕복 비용이 EV에 정확히 반영되는지 고정 시나리오 테스트.
- 판정 게이트: CI 경계 케이스(0 포함/제외) 테스트.

## Phase 1 실측 결과 (2026-07-12)

첫 피검체(기존 패턴 엔진, confirmed 롱) 워크포워드 검증:

- 조건: 2020-01-01 ~ 2026-07-01, 학습 2년/검증 6개월 롤링(검증 구간 2022-01~2026-06),
  유니버스 시총 상위 80 (**current 모드 — 생존 편향 있음**, KRX 로그인 부재로 pit 불가),
  왕복 비용 0.38%, 데이터 커버리지 100% (80/80)
- 결과: 트레이드 2,649건, 거래당 EV **+1.19%** (95% CI +0.84% ~ +1.55%),
  승률 65%, 손익비 0.75, 최악 트레이드 -39.7% (파국적 이상치 없음)
- 랜덤 진입 벤치마크(동일 청산): **+0.66%/거래** — 편향 유니버스+상승장에서는
  아무 데나 사도 벌리는 환경이었다는 정량 증거
- **판정: WATCH** — 원판정은 pass 기준 충족(CI 하한>0, 랜덤 초과)이었으나
  생존 편향 유니버스라 자동 강등. KRX_ID/KRX_PW 설정 후 pit 모드 재검증이 선행 조건.

해석: 기존 패턴 엔진은 이 (유리하게 편향된) 조건에서 랜덤 대비 +0.53%p/거래
우위를 보였다 — 폐기 대상이 아니라 "pit 재검증 대기" 상태가 정확한 위치다.

Phase 1에서 확인된 개선 항목 (Phase 2 반영):
1. MDD 지표가 순차 복리 가정이라 다종목 결과에서 과장됨(100% 표기) →
   포트폴리오 자본곡선(동시 포지션·사이징 반영)으로 교체
2. 런타임 ~2.5시간 → 고정 규칙 전략의 신호 캐시(현재 윈도우당+벤치마크용 중복
   계산), detect_all 병렬화
3. pit 유니버스는 KRX 로그인 필요 — .env KRX_ID/KRX_PW 설정 가이드 문서화
