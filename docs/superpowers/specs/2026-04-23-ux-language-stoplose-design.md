# UX 개선 + 손절/익절 라인 설계

**날짜:** 2026-04-23  
**범위:** 프론트엔드 전체 언어 교체 + 핵심 3페이지 구조 개선 + 차트 손절/익절 라인

## 1. 손절/익절 라인

### 목표
차트에 이미 표시되는 `target_level`(목표가)와 `invalidation_level`(무효화선)의
명칭을 트레이더에게 직관적인 용어로 교체한다.

### 변경 매핑
| 현재 레이블 | 변경 후 | 파일 |
|---|---|---|
| 차트 레전드 "목표가" | "익절 기준가" | `CandleChart.tsx` |
| 차트 레전드 "무효화" | "손절 기준가" | `CandleChart.tsx` |
| AnalysisPanel 내 "무효화 기준" | "손절 기준가" | `AnalysisPanel.tsx` |
| AnalysisPanel 내 "목표가" | "익절 기준가" | `AnalysisPanel.tsx` |
| ChartPage 내 관련 텍스트 | 동일 교체 | `ChartPage.tsx` |

색상은 그대로 유지(초록=익절, 빨강=손절). 선에 가격은 이미 오른쪽 축에 표시됨.

## 3. 대시보드 구조 개선 (DashboardPage.tsx)

### 3-1. 히어로 지표
| 항목 | 현재 | 변경 후 |
|---|---|---|
| 라벨 | 전체 후보 | 오늘 주목할 종목 |
| 힌트 | "N개 즉시 검토, N개 관찰" | "지금 바로 N개 · 대기 중 N개" |

### 3-2. 섹션 이름
| 현재 | 변경 후 |
|---|---|
| 즉시 검토 | 🟢 지금 바로 볼 종목 |
| 관찰 대기 | 🟡 조금 더 기다릴 종목 |
| 리스크 경보 | 🔴 주의 필요 종목 |
| 최근 목표 달성 | ✅ 최근 목표가 도달 |

### 3-3. 종목 카드 한 줄 요약
각 DashboardCard에 패턴 상태 + 오를 확률을 결합한 한 줄 문구 추가.  
예시: `"돌파선 돌파 완료 · 오를 확률 72%"`  
로직: `{STATE_LABELS[state]} · 오를 확률 {p_up}%`  
위치: 종목명 바로 아래, 기존 뱃지 위

## 4. 차트 분석 구조 개선 (AnalysisPanel.tsx + ChartPage.tsx)

### 4-1. 탭 이름 변경
| 현재 | 변경 후 |
|---|---|
| 개요 | 핵심 요약 |
| 셋업 점수 | 진입 준비도 |
| 패턴 상세 | 패턴 분석 |
| 데이터 노트 | 참고사항 |

### 4-2. 핵심 요약 탭 — 상단 현황 카드 추가
탭 최상단에 현황 카드 추가 (기존 내용 아래로 밀림):
```
[패턴명] — [상태]
손절 기준가: XXX원   익절 기준가: XXX원
오를 확률: N%  /  내릴 확률: N%
```
이 카드가 탭 열자마자 가장 먼저 보이도록 배치.
