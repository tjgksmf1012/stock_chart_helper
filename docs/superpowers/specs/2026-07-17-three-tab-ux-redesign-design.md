# 3-탭 여정 재편 — UI/UX 재설계

날짜: 2026-07-17
상태: 승인됨 (사용자 선택: "3-탭 여정 재편 (최대)" + "패턴 후보는 접힌 보조 섹션")

## 문제

앱은 "두 개의 제품이 한 몸"인 상태다.

1. **홈(대시보드) = 옛 패턴 엔진의 세계.** 13개 섹션이 세로로 이어지고, 같은 종목이
   3~4개 섹션에 반복 등장하며, 모든 후보 카드에 동일한 보일러플레이트 문장이 붙는다.
   카드당 지표 7개(상승%·준비도·진입구간·신선도·품질·신뢰도·손익비).
2. **검증을 통과한 실험실 신호(측정된 엣지 +5~7%/거래)는 홈에 한 줄도 없다.**
   내비 9개 중 8번째에 숨어 있고, 들어가도 신호 계산에 1~2분 걸린다.
3. 내비 9개가 평면 나열 — 매일 쓰는 페이지와 참고/관리 페이지가 구분되지 않는다.

랩 검증 결과(패턴 엔진 EV +0.70%/0.04R vs high52 +5.35%/0.67R, tsmom +7.24%/0.47R)와
화면의 면적 배분이 정반대다. 증거 순서와 화면 순서를 일치시킨다.

## 정보 구조

내비 9개 → 여정 3탭 + 유틸 아이콘 2개.

| 새 탭 | 라우트 | 흡수하는 기존 페이지 | 역할 |
|---|---|---|---|
| 오늘 | `/` | 대시보드 + 오늘의 추천 + 실험실 신호 게이트 | 오늘 무엇을 할까 |
| 분석 | `/chart` | 차트 분석 + 종목 필터 + 관심종목 + 패턴 사전 | 이 종목을 파고들기 |
| 기록 | `/journal` | 내 판단·성과 + 종이매매 실측 + 전략 검증 + 패턴 적중률 | 내 성과와 증거 |

- 시스템 상태: 헤더 우측 톱니 아이콘 (`/system` 유지).
- 관심종목 개수 배지: 헤더 별 아이콘 (`/watchlist` = 분석 서브탭).

## 오늘 탭 (신설 TodayPage)

기존 대시보드 13개 섹션 → 5개.

1. **시장 체제 한 줄** — 기존 MarketRegimeBar 압축 재사용. 상세(20/60일선 근거)는 펼치기.
2. **검증된 신호** (최상단 승격) — LabPage의 LiveSignals를 컴포넌트로 추출해 재사용.
   - 신호 행 안에 인라인: 전략 배지(EV), 권장 수량, 손절가, 차트로 이동 링크.
   - 포지션 계산기 설정(계좌·리스크%)은 접힌 설정으로 (localStorage 기존 키 재사용).
   - 계산 중 상태(status:"computing")는 스켈레톤 행 + "1~2분" 안내로 표시.
   - 신호 0건인 날은 "오늘은 검증된 신호 없음 — 없는 날이 정상입니다" 빈 상태.
3. **오늘 확인할 것** — 미정리 판단(장후 닫을 기록) + 관심종목 트리거 근접/손절 위험
   (AiRecommendationsPage의 두 관심종목 섹션 로직 흡수).
4. **관찰 후보 (접힘, 기본 닫힘)** — 기존 후보 섹션 7개(3-3-3, 지금 볼, Live 분봉,
   형성 중, 완성 임박, 교과서 유사, 하락/관망)를 종목 중복 제거 후 하나로 통합.
   - 헤더에 요약: "고유 N종목 · 완성 임박 M".
   - 카드 다이어트: 패턴명+단계 배지, 지표 2개(상승확률·준비도), 종목 고유 문장 1개만.
     전 카드 공통 보일러플레이트("일봉 기준 기준선 재접근...", "핵심 가격대에서...")는 제거.
   - 부제로 정직한 컨텍스트: "검증 엣지 얇음(+0.7%/거래) · 참고용".
   - 정렬 필터(완성 임박순/상승확률순)는 유지하되 탭이 아닌 셀렉트.
5. **스캔 스트립** — 진행 중일 때만 한 줄(진행률+취소). 완료 시 숨김, 상세는 시스템 페이지.

신호 변화(신규/유지/약화) 섹션과 운용 루틴 섹션은 제거한다 — 카운트 정보는
관찰 후보 헤더에 흡수, 루틴 정보는 실사용 가치 대비 면적이 컸다.

## 분석 탭 (AnalysisShell)

서브탭: **[차트 | 종목 필터 | 관심종목 | 패턴 사전]** — 기존 라우트
(`/chart`, `/screener`, `/watchlist`, `/library`)를 그대로 렌더하고 셸만 씌운다.
각 페이지 내용 변경은 차트 페이지만:

- 위: 차트 + 실전 판단 + 확률적 전망(OutlookCard) + 데이터 준비도.
- 아래 접힘(아코디언, 기본 닫힘): 정밀 분석, 수급(MoneyFlowCard), 과거 레퍼런스, 읽는 포인트.
- 패턴명 표시 지점에서 패턴 사전 해당 항목으로 링크.

## 기록 탭 (JournalShell)

서브탭: **[내 기록 | 실측 (종이매매) | 전략 검증 | 패턴 적중률]**

- **내 기록**: 대시보드의 미정리 판단 + 내 성과 요약 이동.
- **실측**: `/lab/paper-trades/summary` 기반 — 전략별 실측 EV·표본·드리프트 상태.
- **전략 검증**: LabPage의 ReportCard들 + "읽기 전 주의" 이동. `/lab` → 이 서브탭으로 리다이렉트.
- **패턴 적중률**: PatternPerformancePage 내용 (`/reports/patterns` 유지).

## 라우팅·마이그레이션

| 기존 | 처리 |
|---|---|
| `/` | TodayPage로 교체 |
| `/ai` | `/`로 리다이렉트 (기능은 오늘 탭에 흡수, AiRecommendationsPage 삭제) |
| `/chart`, `/chart/:symbol` | 분석 셸 안에서 유지 |
| `/screener`, `/watchlist`, `/library` | 분석 셸 서브탭으로 유지 |
| `/lab` | `/journal/strategies`로 리다이렉트 |
| `/reports/patterns` | 기록 셸 서브탭으로 유지 (`/journal/patterns` 별칭) |
| `/journal`, `/journal/:tab` | 신설 |
| `/system` | 유지, 헤더 톱니로 진입 |
| `/reference-charts` | 유지 (차트에서 새 창) |

## 하지 않는 것

- 백엔드 API 변경 없음 (전부 기존 엔드포인트 재사용).
- 기능 삭제 없음 — 전부 재배치·압축. 유일한 페이지 소멸은 오늘의 추천(흡수).
- 다크/라이트 테마, 디자인 토큰 체계 변경 없음.

## 구현 단계 (단계마다 커밋)

1. 셸/라우팅: Layout 3탭 + 유틸 아이콘, 리다이렉트, AnalysisShell/JournalShell 골격.
2. 오늘 탭: TodayPage + LiveSignals 추출·인라인 사이징 + 오늘 확인할 것.
3. 관찰 후보 통합·카드 다이어트 (대시보드 후보 로직 재사용, 중복 제거).
4. 기록 탭 조립 + `/lab`·`/ai` 리다이렉트 + AiRecommendationsPage 제거.
5. 차트 페이지 아코디언 정리 + 전체 QA(데스크톱/모바일, 콘솔, 리다이렉트) + 죽은 코드 제거.
