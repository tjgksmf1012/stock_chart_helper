# 사전 등록 실험 ④ — 외국인 수급 필터: 차단됨 (데이터 접근 불가)

날짜: 2026-07-20
상태: **차단** — 검증용 수급 이력 데이터를 구할 수 없어 실행 불가. 등록만 기록.

## 사전 등록 (실행 전 기록)

- 가설: high52_breakout 신호 중 "그 종목의 외국인 순매수 20일 누적 > 0"인
  신호만 인정하면 EV가 개선된다 (한국 시장에서 반복 보고된 외국인 플로우 팩터).
- 기준선: 공식 게이트 리포트 high52 (+5.55%, CI 하한 +3.15%).
- 판정 문턱: EV +0.5%p 이상 **그리고** CI 하한 개선 시 채택 후보.
- 결측 처리: 수급 데이터 없는 신호는 제외(모르면 안 산다) + 제외 수 기록.
- 인과성: 신호일까지의 수급만 사용 (20일 rolling, 신호일 포함 — 종가 신호와 동일 시점).

## 차단 사유 (2026-07-20 실측)

백테스트(2020~2026, 131종목)에는 일별 투자자 수급 **이력**이 필요한데:

1. **pykrx 투자자 API** — KRX 데이터서비스가 로그인 필수화되어 빈 응답
   (`get_market_trading_value_by_date` 실측: Expecting value 에러, 빈 DataFrame).
   `money_flow_service.py:154`에 이미 기록된 제약과 일치.
2. **KIS 투자자별 매매동향** (`FHKST01010900`) — 최근 ~30영업일만 제공.
   차트 페이지 수급 카드(실시간 표시)에는 충분하지만 백테스트에는 불가.

## 해제 조건

`backend/.env`에 KRX 데이터서비스 계정(`KRX_ID`, `KRX_PW`)이 설정되면
pykrx 로그인 경로로 이력 수집이 가능해진다 — 계정 설정은 사용자가 직접 해야
하며(자격 증명은 코드·대화에 남기지 않는 것이 원칙), 설정 후 "실험 ④ 재개"
한마디면 수집 스크립트부터 이어서 진행한다.

## 실험 프로그램 최종 성적 (2026-07-18~20)

| 실험 | 결과 |
|---|---|
| ① 시장 체제 게이트 | **채택** — 4전략 전부 EV 개선, 전 파이프라인 적용 |
| ② ATR 적응 손절 | 기각 — 전략별 정반대 반응, CI 겹침 |
| ③ 전략 결합 (동일 리스크) | 기각 — 합산 MDD 46.1% > 개별 |
| ③-b 결합 샤프 | 기각 — +0.025 < 문턱 +0.1 (분산 효과 실재하나 미미) |
| ④ 외국인 수급 필터 | 차단 — KRX 로그인 필요 |

교훈: 개인 데이터 환경에서 실재하는 개선은 "언제 안 살지"(체제) 하나였고,
나머지는 그럴듯해 보여도 검증대에서 살아남지 못했다. 4개를 돌려 1개 채택 —
이 비율이 정상이며, 실험실이 확인 사살 기계가 아니라는 증거다.
